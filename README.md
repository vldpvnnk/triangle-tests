# Triangle Validator QA Automation

Проект с автоматизированными UI-тестами для веб-валидатора треугольников.
Выполнен в рамках тестового задания на позицию **QA Engineer**.

## Описание задачи
Необходимо автоматизировать проверку бизнес-логики приложения, которое определяет тип треугольника по длинам сторон.
**Объект тестирования:** [Playground Triangle Puzzle](https://playground.learnqa.ru/puzzle/triangle)

##  Технический стек
В соответствии с требованиями вакансии выбран следующий стек:
- **Язык:** JavaScript (Node.js)
- **Платформа:** Selenium WebDriver
- **Фреймворк:** Mocha
- **Assertion library:** Chai
- **Паттерны:** Retry Logic, Smart Waits

## Покрытие тестами
Реализовано **12 сценариев**, покрывающих позитивные и негативные кейсы:

### 1. Functional Tests (Позитивные)
Проверка корректного определения всех поддерживаемых типов треугольников:
- Прямоугольный, Тупоугольный, Остроугольный
- Равносторонний, Равнобедренный

### 2. Negative Tests (Негативные)
Валидация реакции системы на некорректные данные:
- XSS/Инъекции (ввод букв, спецсимволов)
- Логические ошибки (отрицательные числа, несуществующие треугольники)
- Валидация обязательности полей (пустые значения)

> **Примечание:** Кейсы с SQL и XSS инъекциями были взяты из раздела "Я сдаюсь" на тестируемом сайте. Там был представлен разбор, и эти кейсы были добавлены, чтобы полностью покрыть приложение.

### 3. Edge Cases & Bugs
Сценарии, выявляющие дефекты валидации дробных чисел.

## Найденные дефекты (Bug Report)
В ходе автоматизации были выявлены следующие баги:
1. **TC-016:** Приложение некорректно обрабатывает ввод дробных чисел через точку (считает их НЕ треугольником).
2. **TC-017:** Приложение не принимает дробные числа через запятую.
3. **TC-018:** Приложение не валидирует поле С, если его оставить пустым.
4. **TC-019:** При вводе всех нулей, считается что треугольник равносторонний.
